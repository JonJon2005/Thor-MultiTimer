CURRENT MAINACTIVITY CONTEXT
MainActivity.kt currently contains the full first-pass application UI and most of the app logic:
- MainActivity extends ComponentActivity
- onCreate calls enableEdgeToEdge()
- onCreate calls setContent { FirstAppTheme { TimerApp() } }
- TimerApp is the main top-level composable
- Timer state is stored in a remembered mutableStateListOf<TimerItem>()
- TimerItem currently contains:
  - id
  - totalSeconds
  - remainingSeconds
  - isRunning
  - endTimeMillis
- Timer countdown updates are driven by a LaunchedEffect loop
- The loop recomputes remaining time for running timers based on System.currentTimeMillis()
- Running timers stop automatically when remaining time reaches 0

CURRENT TIMER LOGIC DETAILS
- addTimer(totalSeconds, startImmediately = false):
  - creates a timer
  - inserts it at the front of the list
  - resets currentPage to 0 so newly added timers are visible immediately
- Quick preset buttons call addTimer(..., startImmediately = true)
- Custom timer dialog calls addTimer(..., startImmediately = false)
- Pause logic:
  - computes remaining time from endTimeMillis
  - stops the timer
  - clears endTimeMillis
- Start / Restart logic:
  - restarts from totalSeconds if remainingSeconds == 0
  - otherwise resumes from current remainingSeconds
- Reset logic:
  - returns timer to its original totalSeconds
  - clears running state
- Remove logic:
  - removes the timer from the mutable list
